# Stage 2a ODE integrator — CR3BP benchmark (2026-05-21)

Cross-method, cross-controller benchmark on the shared planar Earth–Moon
CR3BP propagation fixture. Run by `benchmarks/bench_ode_cr3bp` and timed
with Google Benchmark.

## Problem

Planar CR3BP, Earth–Moon mass ratio `μ = 0.01215`. State is
`Eigen::Matrix<double, 4, 1> = (x, y, vx, vy)` in the synodic rotating
frame. IC chosen so the trajectory transits the L1 neck, passes near the
Moon, and exits via L2 over `t ∈ [0, 7]` non-dimensional time units (see
`tests/ode/cr3bp_problem.hpp`).

Every combination is integrated at `abstol = reltol = 1e-12`. Reference
trajectory: `Fehlberg78Integrator + controllers::I` at `1e-14`. Each row
reports wall time (Google Benchmark `cpu_time`) and the endpoint error
`‖x_method(tmax) − x_ref(tmax)‖` against that reference.

## Results

Sorted by wall time ascending within each block. `err` is in
non-dimensional units; `p` = pico (1e-12), `n` = nano (1e-9), `f` = femto
(1e-15).

### Reference baseline

| Combination | Time | err |
|---|---:|---:|
| **Fehlberg78 + I @ 1e-12** (classical baseline) | **48.3 µs** | 141 pm |

The reference itself is timed at `1e-12` but uses the reference trajectory
that was computed at `1e-14`, so `err` is non-zero (it reflects the gap
between the user-tolerance run and the high-precision reference).

### RK methods with PI controller

| Combination | Time | err | Speedup vs ref |
|---|---:|---:|---:|
| Verner 8(7) + PI    | 59.7 µs | 20.6 pm  | 0.81× |
| Verner 9(8) + PI    | 82.3 µs | 4.6 pm   | 0.59× |
| Feagin 12(10) + PI  | 174 µs  | 14.4 pm  | 0.28× |
| Feagin 14(12) + PI  | 250 µs  | 69.2 pm  | 0.19× |

Verner78 is the closest competitor to the Fehlberg78 baseline, with ~25×
better endpoint precision; Verner89 is the smallest-error RK option on
this problem at this tolerance, achieving 4.6 pm error at the cost of ~1.7×
the wall time of the baseline.

### RK methods with H211b smoothing controller

| Combination | Time | err |
|---|---:|---:|
| Verner 9(8) + H211b | 213 µs | 28.5 pm |
| Feagin 14(12) + H211b | 690 µs | 1.26 nm |

H211b's smoothing increases the step count noticeably on this smooth
problem — PI is the better default. H211b becomes attractive only on
bumpy / piecewise-smooth RHS, which CR3BP is not. Feagin14's interaction
with H211b is the slowest combination measured.

### Taylor method, JorbaZou controller, order sweep

| Order | Time | err |
|---:|---:|---:|
| 8  | 3.23 ms | 753 fm  |
| 10 | 2.77 ms | 11.7 pm |
| 12 | 3.10 ms | 8.5 pm  |
| 16 | 4.71 ms | 2.2 pm  |
| 20 | 6.65 ms | 26.1 pm |
| 24 | 8.81 ms | 18.4 pm |
| 30 | 14.03 ms | 23.4 pm |

The order-vs-cost curve has the expected shape: cost grows roughly
linearly with N (~0.45 ms / order between N=12 and N=30), while endpoint
error is dominated by the abstol-tied step-size controller rather than
truncation. At N=16 the Taylor method achieves 2.2 pm precision — the
tightest among Taylor orders measured — but takes ~98× the wall time of
the Fehlberg78 baseline. The Taylor path's advantage on this problem is
not raw speed but accuracy *guarantees from the polynomial form* (event
location via polynomial-Newton, exact dense queries) — none of which the
benchmark exercises directly.

### Taylor with alternate controllers (representative orders)

| Combination | Time | err |
|---|---:|---:|
| Taylor N=12 + PI    | 3.69 ms  | 4.5 pm  |
| Taylor N=24 + H211b | 48.21 ms | 11.6 pm |

PI on Taylor at N=12 is competitive with JorbaZou (3.69 ms vs 3.10 ms)
and achieves better precision (4.5 pm vs 8.5 pm) — worth considering when
the user is already comfortable with PI semantics from the RK side.

H211b on Taylor at N=24 is dramatically slower (48 ms vs 8.8 ms for
JorbaZou at the same order) because the smoothing damps the step
predictor that Taylor needs for efficient adaptive stepping. **Do not
use H211b with Taylor unless you have a specific reason.**

## Takeaways

- **For CR3BP-like smooth astrodynamics propagation at 1e-12 tolerance,
  Verner 8(7) + PI is the best speed–accuracy trade-off in the RK
  family.** It's ~20% slower than the classical Fehlberg baseline but
  delivers ~7× better endpoint precision.
- **Verner 9(8) + PI is the precision champion** among RK methods at
  this tolerance (4.6 pm), at ~70% extra wall time over Verner78.
- **Feagin 14(12) is more expensive than its precision benefit warrants
  on this problem.** Its sparse error indicator (only stages 2 and N−1
  contribute) doesn't help the PI controller pick optimal steps here.
- **Taylor is 30–100× slower than RK for this problem at this tolerance**
  because the per-step cost (N evaluations of the RHS on TE-valued
  state) is O(N²)-leaning, while RK methods complete a step in O(stages)
  RHS evaluations. Taylor's advantage is qualitative (exact polynomial
  event location, smooth dense output) rather than quantitative speed.
- **Step-size controllers**: PI is the right default. I is the
  robustness baseline (the classical RKF78 + I row shows it works fine).
  H211b only earns its place on problems with non-smooth dynamics.

## Caveats and known limitations

- **Feagin err-floor**: Feagin12 and Feagin14 share a tax-side workaround
  for their sparse error indicator (only `k₂ − k_{n−1}` contributes). When
  that difference underflows to zero on benign steps, the PI controller
  is fed `tol · ε` as the floor to prevent step-size collapse. This is
  documented in the Feagin stepper files; it doesn't affect correctness
  but it does shift the (wall-time, err) point slightly compared to
  SciML's reference implementation.
- **No method-order dense output**: All five RK steppers in Stage 2a Plan
  B use cubic-Hermite as their dense-output fallback (`has_dense_output =
  false`). For tests/benchmarks that don't query dense output, this is
  irrelevant. Stage 2b or later may add Verner's published continuous
  extensions.
- **CPU scaling enabled**: Absolute times have ±10% noise; relative
  comparisons within the same run are reliable.
